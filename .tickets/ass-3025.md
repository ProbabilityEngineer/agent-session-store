# Approve repo identity aliases into sidecar

Add a review/approval workflow for repo identity candidates that writes approved aliases/observations/events into repo-identities.jsonl. Support manual curation for renamed projects such as cypv1, checkyourphotosv1, and check-your-photos-v1 mapping to a single stable project identity. Preserve confidence/provenance and avoid automatic weak merges.

## Acceptance Criteria

Approved candidates can append repo-identity/repo-observation/repo-event records to repo-identities.jsonl; manual entries can define stableName/displayName and multiple paths; generated records include evidence/provenance/confidence; rerunning build attaches sessions to approved repo identity aliases.
